# Ventri i18n Design ÔÇö English + Spanish
Date: 2026-05-11

## Goal
Add Spanish (es) as a supported locale alongside English (en). All user-facing strings must be translatable. No i18n infrastructure exists today; this spec establishes it from scratch.

## Scope
`androidApp` module only. `shared` module is unaffected.

---

## ┬º1 ÔÇö String Resource Files

### Files
| File | Purpose |
|------|---------|
| `androidApp/src/main/res/values/strings.xml` | English default |
| `androidApp/src/main/res/values-es/strings.xml` | Spanish override |

### Naming Convention
`<screen>_<element>` for screen-specific strings. `common_` prefix for shared strings.

| Prefix | Screens / usage |
|--------|----------------|
| `common_` | Cancel, OK, Yes, No, Edit, Remove, Confirm, etc. |
| `overview_` | OverviewScreen |
| `items_` | ItemsScreen + ItemsViewModel events |
| `stock_` | StockScreen + StockViewModel |
| `shopping_` | ShoppingScreen + ShoppingViewModel |
| `prefs_` | PreferencesScreen |
| `notification_` | NotificationHelper |

### Plurals
4 `<plurals>` resources replace inline Kotlin pluralization (`"item${if n != 1 "s" else ""}`). All defined here for reference (┬º3 covers the screen-specific ones in detail):

| Key | Usage |
|-----|-------|
| `item_count` | Items screen header, Stock header, Shopping header |
| `stock_days_remaining` | StockScreen days label |
| `overview_days_remaining` | OverviewScreen `toDaysText()` |
| `shopping_will_last` | ShoppingScreen depletion label |

```xml
<!-- values/strings.xml -->
<plurals name="item_count">
    <item quantity="one">%d item</item>
    <item quantity="other">%d items</item>
</plurals>

<!-- values-es/strings.xml -->
<plurals name="item_count">
    <item quantity="one">%d art├¡culo</item>
    <item quantity="other">%d art├¡culos</item>
</plurals>
```

Usage in Compose:
```kotlin
pluralStringResource(R.plurals.item_count, count, count)
```

### SortOrder Labels
`SortOrder` enum currently has `label: String` ÔÇö remove this field. `ItemsScreen` maps each value to `stringResource()`:
```kotlin
val label = when (order) {
    SortOrder.Priority -> stringResource(R.string.items_sort_priority)
    SortOrder.NameAsc -> stringResource(R.string.items_sort_name_asc)
    SortOrder.NameDesc -> stringResource(R.string.items_sort_name_desc)
}
```

## ┬º2 ÔÇö ViewModel Event Types

Replace `SharedFlow<String>` error/event flows with typed sealed classes. UI does all `stringResource()` mapping. ViewModels remain Android-context-free.

### ItemsViewModel

```kotlin
// file: ui/items/ItemsEvents.kt
sealed interface InsertItemError {
    object DuplicateName : InsertItemError
    data class LimitReached(val limit: Int) : InsertItemError
}

sealed interface UpdateItemError {
    object DuplicateName : UpdateItemError
}

sealed interface ItemsSnackbarEvent {
    object AddedToShopping : ItemsSnackbarEvent
    object AlreadyInShopping : ItemsSnackbarEvent
    data class SelectionFailed(val count: Int) : ItemsSnackbarEvent
}
```

ViewModel flow types change:
- `_addItemResult: MutableSharedFlow<InsertItemError?>` (null = success)
- `_editItemResult: MutableSharedFlow<UpdateItemError?>` (null = success)
- `_snackbarMessage: MutableSharedFlow<ItemsSnackbarEvent>`

### OverviewViewModel

```kotlin
// file: ui/overview/OverviewEvents.kt
sealed interface OverviewError {
    object AddToShoppingListFailed : OverviewError
}
```

`_errors: MutableSharedFlow<OverviewError>`

### VentriApplication + AddToShoppingListReceiver + ShoppingViewModel

```kotlin
// file: ui/shopping/ShoppingEvents.kt
sealed interface ShoppingError {
    data class AddFailed(val cause: String?) : ShoppingError
}
```

`VentriApplication.pendingShoppingError: MutableStateFlow<ShoppingError?>` (was `String?`).
`AddToShoppingListReceiver` already has `Context` ÔÇö it emits a typed error instead of a string.
`ShoppingViewModel.pendingError: StateFlow<ShoppingError?>`.
`ShoppingScreen` maps the typed error to a `stringResource()` title/message for the dialog.

## ┬º3 ÔÇö Computed Label Types in UiModels

Two UiModel fields currently hold pre-built strings built inside ViewModels. Replace with typed sealed classes; Compose does the string mapping.

### StockViewModel / StockItemUiModel

```kotlin
// file: ui/stock/StockLabels.kt
sealed interface StockDaysLabel {
    object TrackingUsage : StockDaysLabel
    object LessThanADay : StockDaysLabel
    data class Days(val count: Long) : StockDaysLabel
}
```

`StockItemUiModel.daysRemainingLabel: String` ÔåÆ `StockItemUiModel.daysLabel: StockDaysLabel`

`StockScreen` maps:
```kotlin
val text = when (val label = item.daysLabel) {
    StockDaysLabel.TrackingUsage -> stringResource(R.string.stock_tracking_usage)
    StockDaysLabel.LessThanADay -> stringResource(R.string.stock_less_than_a_day)
    is StockDaysLabel.Days -> pluralStringResource(R.plurals.stock_days_remaining, label.count.toInt(), label.count)
}
```

### ShoppingViewModel / ShoppingItemUiModel

```kotlin
// file: ui/shopping/ShoppingLabels.kt
sealed interface ShoppingDepletionLabel {
    object AlreadyDepleted : ShoppingDepletionLabel
    data class WillLast(val days: Long) : ShoppingDepletionLabel
}
```

`ShoppingItemUiModel.depletionLabel: String?` ÔåÆ `ShoppingItemUiModel.depletionLabel: ShoppingDepletionLabel?`

`ShoppingScreen` maps:
```kotlin
val text = when (val label = item.depletionLabel) {
    null -> null
    ShoppingDepletionLabel.AlreadyDepleted -> stringResource(R.string.shopping_already_depleted)
    is ShoppingDepletionLabel.WillLast -> pluralStringResource(R.plurals.shopping_will_last, label.days.toInt(), label.days)
}
```

### OverviewScreen.toDaysText()

Already inside a Compose context. Make it a `@Composable` extension so it can call `stringResource()` directly:

```kotlin
@Composable
private fun Long.toDaysText(): String {
    val days = this / MILLIS_PER_DAY
    return when {
        this <= 0 && days == 0L -> stringResource(R.string.overview_depleting_today)
        this < MILLIS_PER_DAY -> stringResource(R.string.overview_less_than_a_day)
        days == 1L -> stringResource(R.string.overview_one_day)
        else -> pluralStringResource(R.plurals.overview_days_remaining, days.toInt(), days)
    }
}
```

## ┬º4 ÔÇö Notification Strings

`NotificationHelper` uses hardcoded English strings. Replace with `context.getString()`:

```kotlin
// Before
"Critical Stock Alerts"
"Daily alerts when items are critically low on stock"
"Stock Alert"
"Shopping List Updated"

// After
context.getString(R.string.notification_channel_name)
context.getString(R.string.notification_channel_description)
context.getString(R.string.notification_stock_alert_title)
context.getString(R.string.notification_shopping_updated_title)
```

## ┬º6 ÔÇö Invariants

```
V1: ÔêÇ user-facing string ÔåÆ in strings.xml (not hardcoded in .kt)
V2: ÔêÇ ViewModel event ÔåÆ typed sealed class (no String in SharedFlow)
V3: ÔêÇ UiModel label field ÔåÆ typed sealed class (no String for computed labels)
V4: plurals ÔåÆ <plurals> resource (no inline Kotlin s/es suffix logic)
V5: no Android Context Ôêê ViewModel (stringResource only in Compose/Context owners)
```
